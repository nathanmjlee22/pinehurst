# Pinehurst 2026 — Claude Context

## What this project is
A golf trip dashboard for 9 players attending Pinehurst Resort in September 2026. Hosted on GitHub Pages at `nathanmjlee22/pinehurst`.

## Build pipeline
```
fetch_ghin.py  →  ghin_data.json  →  build_page.py  →  index.html / handicap.html
```
- `fetch_ghin.py` logs into GHIN API (`api.ghin.com/api/v1`) using `USER` / `PASSWORD` env vars and writes `ghin_data.json`
- `build_page.py` reads `ghin_data.json` and writes `index.html` and `handicap.html` (identical content)
- GitHub Actions (`refresh.yml`) runs the pipeline daily at 3 AM PST and on manual dispatch

## HTML templating rule
The HTML template in `build_page.py` is a **plain Python string**, not an f-string. CSS/JS braces would conflict. Substitutions use `__PLACEHOLDER__` tokens replaced via `.replace()` at the bottom of the file.

## Key files
| File | Purpose |
|------|---------|
| `fetch_ghin.py` | GHIN API client — fetches handicap history + tournament scores |
| `build_page.py` | Page generator — contains full HTML/CSS/JS template |
| `ghin_data.json` | Fetched GHIN data (committed, regenerated by workflow) |
| `scores.json` | Manually entered match results + tee selections (synced via GitHub Contents API) |
| `putterboy.png` | Pinehurst logo used in the header |
| `.github/workflows/refresh.yml` | GitHub Actions — daily refresh + manual trigger |

## Scores sync
Match results and tee selections are stored in `scores.json` and written directly to GitHub via the Contents API (no workflow needed). The "💾 Save Scores" button in the UI triggers `pushScores()` which PUTs to `scores.json`. A GitHub PAT with `contents: write` scope is stored in `localStorage` on each device.

## Feature flags in build_page.py
| Flag | Default | Effect |
|------|---------|--------|
| `SHOW_REFRESH_BTN` | `False` | Shows/hides the "↻ Refresh GHIN" button in the header |

## Course handicap formula
USGA 2020: `round(HI × (Slope / 113) + (Rating − Par))`  
Stroke allocation: lowest course handicap = scratch, everyone else gets the difference.

## After editing build_page.py
Always run `python build_page.py` then check JS syntax:
```bash
python3 -c "
import re, subprocess
content = open('index.html').read()
scripts = re.findall(r'<script(?! src)[^>]*>(.*?)</script>', content, re.DOTALL)
for i, s in enumerate(scripts):
    open('/tmp/test_script.js','w').write(s)
    r = subprocess.run(['node','--check','/tmp/test_script.js'], capture_output=True, text=True)
    print(f'Script {i}:', 'OK' if r.returncode == 0 else r.stderr)
"
```
